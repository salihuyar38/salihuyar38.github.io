Building an Elastic Task Queue with MPI

July 2025

Introduction

High-Performance Computing (HPC) refers to the use of multiple computing resources working together to solve computational problems that are too large or time-consuming for a single machine. One of the most common programming models in HPC is the Message Passing Interface (MPI), which allows processes running on different nodes to communicate and coordinate their work.

Real-world HPC systems are significantly more complex than the project presented here. The goal of this project was to explore concepts such as dynamic task distribution, load balancing, and message passing at a beginner-friendly level.

Project Overview

In this project, I implemented a lightweight elastic task queue using MPI. A master process dynamically distributes tasks to worker processes during runtime. Workers request tasks, process them, and return their results to the master.

This approach improves load balancing because faster workers naturally process more tasks than slower workers.

Failure Experiment

To better understand distributed systems, I also experimented with terminating a worker process during execution and observed how the application behaved under failure conditions.
